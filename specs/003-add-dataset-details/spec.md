## Clarifications

### Session 2026-03-15

- Q: セクション内で動画サンプルをどう扱うか？ → A: 動画は使用せず、表を用いて構造を説明する
- Q: セクションの見出し階層と幅の扱いはどうするか？ → A: 既存ページの同種セクションと同じ見出し階層・グリッド幅に合わせる
- Q: 表の情報提示順はどうするか？ → A: 「サブセット→動画/注釈→メタデータ」の流れ
- Q: 想定読者の区分はどうするか？ → A: 「研究者・一般来訪者」の2区分のみ
- Q: 欠損値の意味をどう示すか？ → A: 欠損値の意味を注記で明記する

## User Scenarios & Testing *(mandatory)*

### User Story 1 - データ構造をひと目で理解する (Priority: P1)

研究者・一般来訪者として、ub-MOJI のデータ構造（サブセット・ファイル形式・メタデータ）を短時間で把握し、研究への適合性を判断したい。

**Why this priority**: データ構造の理解が最初の判断材料であり、閲覧者の離脱を防ぐために最重要。

**Independent Test**: 詳細説明セクションのみを見て、サブセット構成・主要ファイル・命名規則を説明できる。

**Acceptance Scenarios**:

1. **Given** 初回訪問のユーザーがいる, **When** 詳細説明セクションを見る, **Then** `syllables` / `sequences` / `words` の3構成とそれぞれの内容が明確に示される
2. **Given** ユーザーが動画データの形式を知りたい, **When** セクションを見る, **Then** `.mp4` の動画であることが視覚的に分かる

---

### User Story 2 - メタデータと注釈の意味を理解する (Priority: P2)

研究者として、メタデータと注釈ファイルの役割や含まれる項目を把握し、分析設計の前提を整えたい。

**Why this priority**: データの再利用性と分析可能性の理解に直結するため。

**Independent Test**: セクションだけで `metadata.csv` / `participants.csv` / `annotations.toml` の役割と代表項目を説明できる。

**Acceptance Scenarios**:

1. **Given** メタデータの概要を知りたいユーザーがいる, **When** セクションを見る, **Then** サンプル単位・参加者単位・時間注釈の違いが区別できる

---

### User Story 3 - 利用条件の制約を把握する (Priority: P3)

一般来訪者として、データセットの利用制約（学術目的限定・再配布禁止など）を理解し、利用可否を判断したい。

**Why this priority**: 利用条件の誤解は重大なリスクを生むため、可視化された説明が必要。

**Independent Test**: セクションだけで主要な利用条件を言い当てられる。

**Acceptance Scenarios**:

1. **Given** 研究目的で利用検討中のユーザーがいる, **When** セクションを見る, **Then** 学術目的限定・商用不可・再配布不可が簡潔に示される

---

### Edge Cases

- データセットの一部が公開同意を得ていない場合、注記が分かるか？
- メタデータ項目の一部が未提供（例: `-1`）の場合、その意味が注記で明確に伝わるか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは ub-MOJI の3サブセット（`syllables` / `sequences` / `words`）の違いを一目で理解できる視覚説明を提供しなければならない
- **FR-002**: システムは動画データ形式（RGB `.mp4`）と注釈ファイル（`.toml`）の関係性を、動画を表示せず表で示さなければならない
- **FR-003**: システムはファイル命名規則（`{content}_{participantID}_{yyyymm}_{take}.mp4`）を図解で説明しなければならない
- **FR-004**: システムは `metadata.csv` / `participants.csv` / `annotations.toml` の役割と主要項目を、テキストと可視化の組み合わせで説明しなければならない
- **FR-005**: システムは利用条件（学術目的限定、商用不可、再配布不可）をセクション内で明確に提示しなければならない
- **FR-006**: システムは既存ページの同種セクションと同じ見出し階層・グリッド幅に合わせ、色・タイポグラフィ・余白・図表スタイルが統一されていることをレビューで確認できる状態で提示しなければならない
- **FR-007**: システムは表の情報提示順を「サブセット→動画/注釈→メタデータ」の順に構成しなければならない
- **FR-008**: システムは欠損値（例: `-1`）の意味を注記で明確に示さなければならない

### Key Entities *(include if feature involves data)*

- **Dataset Section**: ub-MOJI のデータ構造と利用条件を説明するページ内セクション
- **Subset**: `syllables` / `sequences` / `words` の各データ単位
- **Video Sample**: RGB `.mp4` の動画サンプル
- **Metadata**: `metadata.csv` / `participants.csv` の項目情報
- **Annotation**: `annotations.toml` に記載される時間注釈

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 初回閲覧者の70%以上が、3サブセットの違いを30秒以内に説明できる
- **SC-002**: 80%以上のユーザーが、メタデータと注釈の役割を誤解なく言い当てられる
- **SC-003**: セクション閲覧後の「データ構造の理解度」自己評価が平均4.0/5以上になる
- **SC-004**: 利用条件の主要3点を正しく理解できたユーザー割合が90%以上になる

## Dependencies

- Hugging Face の ub-MOJI データセットカードに記載された最新の構造情報
- 対象ページの既存デザインガイドライン（色・タイポグラフィ・余白・図表スタイル）

## Assumptions

- 参照元は Hugging Face の ub-MOJI データセットカードであり、そこに記載された構造・用語を基準にする
- 詳細説明セクションは既存のデータセット紹介ページに追加する
- 表現は日本語を基本とし、専門用語は必要最小限に留める
- 本機能の範囲は「詳細説明セクションの追加と内容提示」に限定し、ダウンロード導線や権限管理の変更は含めない
